The Harding Computer Science department fundraiser is a contest in which people guess how many jelly beans are in a jar. Each person may play multiple times, and therefore make multiple guesses. You are to create a web service which will store these guesses in a MongoDB database and retrieve them upon request.

  1. Your config module should export a db variable which is an object containing the following properties.
    • host — the computer the database server is on
    • user — the username to connect with
    • pass — the password to connect with
    • name — the name of the database to use
    I should be able to use your application with my database simply by changing the value of these properties.
  2. Create a Mongoose model type to represent guesses. Each guess should have a user (string) and a count (number).
  3. To create a new record you should make a POST request to /api/jellybeans/. The content of the request should specify the user and the guess. 
    { "user" : "gfoust",
  "count" : 230
}
  4. A GET request to /api/jellybeans/ should retrieve all guesses as a JSON array. 
    [ { "user" : "gfoust",
    "count" : 230
  },
  { "user" : "jsmith",
    "count" : 258
  },
  { "user" : "mjones",
    "count" : 300
  },
  { "user" : "gfoust",
    "count" : 270
  }
]
  5. A GET request to /api/jellybeans/from-to should retrieve all guesses where count is between from and to as a JSON array. 
    /api/jellybeans/255-275
    [ { "user" : "jsmith",
    "count" : 258
  },
  { "user" : "gfoust",
    "count" : 270
  }
]
  6. The url /api/ should show my web-service page to test your web services.
